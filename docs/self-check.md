# 项目自检报告

## 当前定位

本项目已经从普通英语阅读练习页，升级为偏考试训练的英语阅读理解系统。当前包含大量题库、分类筛选、计时、自动评分、混合题型、PWA 部署和移动端适配。

## 已发现并修复的问题

### 1. 每篇文章题量过载

此前系统给每篇文章追加了大量题型，导致单篇文章可能出现几十道题。虽然题型丰富，但用户体验不符合真实考试训练。

已修复：新增 `js/practice-mode-runtime.js`，每次练习从完整题库中抽取 18 道代表性考试题，覆盖判断、匹配、摘要填空、流程图、简答、证据定位等题型。

### 2. 题型偏普通练习

此前虽然题型多，但部分仍偏普通训练题。

已修复：新增 `js/exam-question-types.js`，加入 IELTS / TOEFL / CET 风格题型，包括 True / False / Not Given、Yes / No / Not Given、Matching Headings、Matching Information、Sentence Insertion、Rhetorical Purpose、Summary Completion、Flow-chart Completion 等。

### 3. UI 过于工具化

此前页面偏功能堆叠，视觉不够像正式学习 App。

已修复：新增 `css/polished-ui.css`，优化 Hero、分类卡片、筛选区、文章卡片、阅读页、结果页、移动端布局。

### 4. 缓存更新容易滞后

此前 Service Worker 使用缓存优先，容易导致用户刷新后仍看到旧版。

已修复：`sw.js` 已切换为网络优先，离线时才回退缓存，并更新缓存版本为 `english-reading-trainer-v12-selfcheck-balanced`。

## 仍然存在的不足

### 1. 文章和题目内容仍然偏模板化

虽然数量很大，但基础大题库和扩展题型依赖模板生成，内容真实度和题目针对性不如人工精选题库。

建议下一步：建立 300-1000 篇真正高质量人工精选题库，按难度和考试类型分层。

### 2. 考试模式还不完整

目前是考试型混合题组，但还没有完整模拟：

- IELTS Reading 3 passages / 40 questions / 60 minutes
- TOEFL Reading 10 questions per passage / fixed time
- CET / 高考阅读模块化训练

建议下一步：增加考试模式选择器。

### 3. 简答题评分仍是关键词评分

当前无 AI API，因此简答、摘要、改写只能用关键词命中率评分，不能真正判断语义质量。

建议下一步：增加本地规则评分说明，或者以后接入本地模型 / 后端评分。

### 4. 缺少错题本

当前只保存历史分数，没有按题目保存错题。

建议下一步：保存错题、题型、用户答案、正确答案、解析，并支持按题型复习。

### 5. 缺少词汇本

阅读训练最需要词汇积累。目前还没有生词收藏和复习。

建议下一步：加入词汇高亮、手动收藏、词义备注、复习列表。

### 6. 缺少账号同步

当前历史记录保存在浏览器 localStorage。换设备或清缓存会丢失。

建议下一步：接入 Supabase / Firebase，支持登录、云同步、排行榜。

### 7. 题型筛选还不够细

目前可以按题库、难度、主题筛选，但不能按题型筛选。

建议下一步：增加题型筛选，例如只练 Matching Headings、只练 True / False / Not Given。

## 推荐下一轮优先级

1. 增加考试模式选择器：IELTS / TOEFL / CET / 高考 / 自由练习。
2. 增加错题本。
3. 增加题型专项训练。
4. 增加词汇本。
5. 提高精选题库质量，而不是继续盲目扩大数量。
